1. Introduction

After installing the ContentConnect module, it is recommended to test the individual components.

This document describes such tests.

These test instructions assume that the ContentConnect module has already been installed and configured with all necessary components. The necessary installation and configuration steps are explained in the ContentConnect Documentation.

2. Project component

To check the configuration of the project component open the ServerManager and select the Project properties → Project Components area. A list of all existing project components is displayed in the main panel. Select the entry ContentConnect Project Configuration and open the corresponding configuration dialog via Configure:

Figure 1. Configuration dialog of the project component


The button Test Configuration allows to check the entered entries. In case of a failed test check if the Commerce Cloud instance is reachable and if the Client ID, the Base URL and - if available - the Image Service Base URL are correct.

3. Preview

Open the project in ContentCreator and judge whether the storefront is displayed correctly:

The correct display of the preview shows the storefront of Salesforce as well as the grey areas which can be used to create content in FirstSpirit.

Figure 2. Correct display of the homepage


In case of an error, the preview will only show the grey areas from FirstSpirit, while the content coming from the storefront is missing. Instead, the preview displays unformatted text with regex definitions.

Figure 3. Incorrect display of the homepage


If, in addition to the incorrect display of the preview, the reports are missing, the ContentConnect module is not installed correctly.

Otherwise, an incorrect definition of the project settings in the SiteArchitect or an incorrect configuration of the web.xml are possible causes for the incorrect preview display.

4. Reports

Open the ContentCreator to check the availability of the reports. If the icons of both reports are missing on the right side, the ContentConnect module has not been installed properly. If only the icon of the product report is visible, the checkbox Show Category Report in the project component is disabled.

Figure 4. Report icons


The category report displays a list of available categories as soon as it is opened. In contrast, in the product report, you must first select a category or enter a search term to obtain a corresponding list of products.

If one of the two reports does not show any results, please check the Base URL and the login data in the project component.

Figure 5. Category and product report


The button allows you to create a new category or product detail page. It is displayed by hovering over an entry. If an error message appears when clicking the button, check the configuration of the Category Page Template Mapping or Product Page Template Mapping in the project component.

Figure 6. Incorrect reference


5. Deployment

The transfer of the XML into the Commerce Cloud is done by a deployment. This requires the release of a page maintained in FirstSpirit and the start of the Publish workflow. For example, edit the product and category pages created in the previous step, release them and then publish them using the workflow. This can be triggered with the action Publish in the ContentCreator.

Figure 7. Publish workflow in the ContentCreator


Then open the Salesforce storefront and navigate to the pages you have previously edited in FirstSpirit. They must now also display content originating from FirstSpirit on Salesforce-side. If this is not the case, check the settings of the deployment job in the Project properties → Schedule management area within the ServerManager and the Job Schedules in the Commerce Cloud.
