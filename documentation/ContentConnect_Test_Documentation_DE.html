1. Einleitung

Nach der Installation des ContentConnect-Moduls ist ein Test der einzelnen Komponenten empfohlen.

Das vorliegende Dokument beschreibt solche Tests.

Diese Testanleitung setzt voraus, dass das ContentConnect-Modul bereits mit allen nötigen Komponenten installiert und konfiguriert wurde. Die dafür notwendigen Installations- und Konfigurationsschritte sind in der ContentConnect-Dokumentation erläutert.

2. Projekt-Komponente

Öffnen Sie für die Überprüfung der Konfiguration der Projekt-Komponente den ServerManager und wählen Sie den Bereich Projekt-Eigenschaften → Projekt-Komponenten. Im Hauptpanel ist eine Liste aller bereits vorhandenen Projekt-Komponenten zu sehen. Selektieren Sie den Eintrag ContentConnect Project Configuration und öffnen Sie den zugehörigen Konfigurationsdialog über Konfigurieren:

Abbildung 1. Konfigurationsdialog der Projekt-Komponente


Der Button Test Configuration ermöglicht die Überprüfung der eingetragenen Eingaben. Kontrollieren sie im Fall eines fehlgeschlagenen Tests, ob die Commerce Cloud-Instanz erreichbar ist und die Client ID, die Base URL sowie - wenn vorhanden - die Image Service Base URL korrekt sind.

3. Vorschau

Öffnen Sie das Projekt im ContentCreator und beurteilen Sie, ob die Storefront korrekt dargestellt wird:

Die korrekte Darstellung der Vorschau zeigt sowohl die Storefront von Salesforce als auch die grauen Bereiche, über die in FirstSpirit Inhalte angelegt werden können.

Abbildung 2. Korrekte Darstellung der Homepage


Im Fehlerfall zeigt die Vorschau lediglich die grauen Bereiche aus FirstSpirit an, während die aus der Storefront stammenden Inhalte fehlen. Stattdessen stellt sie einen unformatierten Text mit Regex-Definitionen dar.

Abbildung 3. Fehlerhafte Darstellung der Homepage


Fehlen zusätzlich zur fehlerhaften Darstellung der Vorschau die Reports, ist das ContentConnect-Modul nicht richtig installiert.

Andernfalls sind eine fehlerhafte Definition der Projekteinstellungen im SiteArchitect oder eine inkorrekte Konfiguration der web.xml mögliche Ursachen für die falsche Vorschaudarstellung.

4. Reports

Öffnen Sie zur Überprüfung der Verfügbarkeit der Reports den ContentCreator. Fehlen auf der rechten Seite die Icons beider Reports, wurde das ContentConnect-Modul nicht richtig installiert. Ist lediglich das Icon des Produkte-Reports sichtbar, ist die Checkbox Show Category Report in der Projekt-Komponente deaktiviert.

Abbildung 4. Report-Icons


Der Kategorie-Report zeigt, sobald er geöffnet wird, eine Liste verfügbarer Kategorien an. Im Gegensatz dazu muss im Produkte-Report zuerst eine Kategorie ausgewählt oder ein Suchbegriff eingegeben werden, um eine entsprechende Liste von Produkten zu erhalten.

Zeigt einer der beiden Reports keine Ergebnisse an, prüfen Sie bitte die Base URL und die Login-Daten in der Projekt-Komponente.

Abbildung 5. Kategorie- und Produkte-Report


Die Schaltfläche ermöglicht die Erzeugung einer neuen Kategorie- bzw. Produktdetailseite. Sie erscheint beim Hovern über einen Eintrag. Erscheint bei einem Klick auf die Schaltfläche eine Fehlermeldung, überprüfen Sie die Konfiguration des Category Page Template Mappings bzw. Product Page Template Mappings in der Projekt-Komponente.

5. Deployment

Die Übertragung des XMLs in die Commerce Cloud erfolgt durch ein Deployment. Dies erfordert die Freigabe einer in FirstSpirit gepflegten Seite und den Start des Publish-Arbeitsablaufs. Editieren Sie hierfür beispielsweise die im vorherigen Schritt angelegten Produkt- und Kategorieseiten, geben Sie diese frei und publizieren Sie sie anschließend mithilfe des Arbeitsablaufs. Dieser lässt sich über die Aktion Publish im ContentCreator anstoßen.

Abbildung 7. Publish-Arbeitsablauf im ContentCreator


Öffnen Sie danach die Salesforce-Storefront und navigieren Sie dort zu den Seiten, die Sie zuvor in FirstSpirit bearbeitet haben. Sie müssen nun Salesforce-seitig auch die aus FirstSpirit stammenden Inhalte anzeigen. Ist dies nicht der Fall, kontrollieren Sie die Einstellungen des Deployment-Auftrags im Bereich Projekt-Eigenschaften → Auftragsverwaltung innerhalb des ServerManagers sowie die Job Schedules in der Commerce Cloud.
